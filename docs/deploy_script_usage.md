# Mem0自动化部署脚本使用说明

## 简介

`deploy.sh`是一个交互式Bash脚本，提供了菜单驱动的Mem0部署和管理功能。它能帮助您轻松完成Mem0的部署、服务管理、系统监控和备份等操作，无需记忆复杂的命令。

## 功能概览

脚本提供以下主要功能：

1. **Docker部署**
   - 完整模式部署
   - 服务重建（清除旧数据）

2. **开发模式**
   - 使用uvicorn直接启动开发服务器

3. **服务管理**
   - 停止服务
   - 重启服务
   - 查看服务状态
   - 查看日志

4. **系统管理**
   - 系统清理
   - 磁盘空间分析
   - 健康检查
   - 备份管理（创建/恢复/列出备份）

## 使用前提

- 已安装Docker和Docker Compose
- 有适当的系统权限（需要操作Docker和文件系统）
- 可选：安装curl（用于健康检查）

## 快速开始

1. 确保脚本有执行权限：
   ```bash
   chmod +x deploy.sh
   ```

2. 运行脚本：
   ```bash
   ./deploy.sh
   ```

3. 按照屏幕上的菜单提示进行操作

## 环境变量支持

脚本支持以下环境变量：

- `DATA_DIR`: 指定数据存储目录的位置。默认为`./data`

示例：
```bash
DATA_DIR=/path/to/data ./deploy.sh
```

## 菜单导航

脚本使用数字菜单系统进行导航：

1. 在主菜单中选择功能类别（1-5）
2. 在子菜单中选择具体操作
3. 执行操作后按回车返回菜单
4. 选择"返回"选项回到上级菜单

## 数据备份与恢复

### 创建备份

1. 从主菜单选择"Mem0系统管理"
2. 选择"备份管理"
3. 选择"创建备份"
4. 备份文件将保存在`./backups`目录中

### 恢复备份

1. 从主菜单选择"Mem0系统管理"
2. 选择"备份管理"
3. 选择"恢复备份"
4. 选择要恢复的备份文件
5. 确认操作

## 注意事项

- 重新构建服务将删除所有现有数据，请确保先备份重要数据
- 恢复备份会停止服务，恢复数据后再重启服务
- 健康检查功能需要curl命令支持
- 所有数据目录操作都基于`DATA_DIR`环境变量或默认值

## 故障排除

### 依赖检查失败

脚本会自动检查必要的依赖项。如果依赖检查失败：
- 确保已安装Docker: `docker --version`
- 确保已安装Docker Compose: `docker-compose --version`
- 可选：安装curl: `apt-get install curl` 或 `yum install curl`

### 服务启动失败

如果服务无法启动：
1. 检查服务状态: 从菜单选择"服务管理" > "查看服务状态"
2. 查看日志: 从菜单选择"服务管理" > "查看日志"
3. 确保所有必要的端口未被占用

### 备份/恢复问题

如果备份或恢复操作失败：
1. 确保有足够的磁盘空间
2. 检查目录权限
3. 确保备份文件存在且未损坏

## 自定义与扩展

您可以根据需要修改脚本，添加更多功能或自定义现有功能：

- 添加新的菜单选项
- 调整Docker Compose文件路径
- 修改备份策略
- 添加更多系统监控功能
